## 🤔 Зачем это нужно?

3proxy — лёгкий и быстрый многопротокольный прокси-сервер. Этот скрипт разворачивает его за одну команду:

- 🌐 **HTTP / HTTPS прокси** — любое количество портов, каждый настраивается отдельно
- 🧦 **SOCKS5** — опционально, тоже любое количество портов
- 🔐 **Авторизация на каждый порт** — свой логин/пароль для каждого прокси, или открытый режим
- 🛡️ **ACL по IP** — разрешить доступ только с нужных адресов или подсетей
- ⚙️ **systemd** — автозапуск, рестарт при падении, управление как любым сервисом

## ✨ Возможности

### 🌐 Динамическое количество портов

Скрипт не ограничивает количество прокси — после каждого добавленного порта предлагает создать ещё один:

```
  [INFO]  Настройка HTTP-прокси (минимум один порт)

  ──  HTTP-прокси #1  ──────────────────────────────
  ►  Порт HTTP-прокси #1 [10000]: 10000
  [ OK ]  HTTP-прокси #1: порт 10000
  ►  Добавить ещё один HTTP-прокси? [y/N]: y

  ──  HTTP-прокси #2  ──────────────────────────────
  ►  Порт HTTP-прокси #2 [10100]: 10100
  [ OK ]  HTTP-прокси #2: порт 10100
  ►  Добавить ещё один HTTP-прокси? [y/N]: n
```

SOCKS5 полностью опционален — скрипт спрашивает нужен ли он вообще:

```
  ►  Нужен SOCKS5-прокси? [y/N]:
```

Если да — то же самое: порт, авторизация, вопрос «добавить ещё?».

---

### 🔐 Авторизация на каждый порт

Каждый прокси настраивается независимо. Сразу после выбора порта скрипт спрашивает про авторизацию именно для него:

```
  ──  HTTP-прокси #1  ──────────────────────────────
  ►  Порт HTTP-прокси #1 [10000]:
  ►  Включить авторизацию для HTTP-прокси #1 :10000? [Y/n]:
      → Логин: alice
      → Пароль (скрытый ввод): ••••••••
  [ OK ]  Авторизация: пользователь alice

  ──  HTTP-прокси #2  ──────────────────────────────
  ►  Порт HTTP-прокси #2 [10100]:
  ►  Включить авторизацию для HTTP-прокси #2 :10100? [Y/n]: n
  [ OK ]  Авторизация: отключена (открытый прокси)
```

Варианты:
- **С паролем** — логин/пароль, пароль хранится как MD5-хэш в `/etc/3proxy/.proxyauth`
- **Без пароля** — открытый прокси, доступ только через ACL по IP

В конфиге для каждого порта генерируется отдельный блок `auth`:

```
# HTTP-прокси #1 (:10000) — с авторизацией
flush
users /etc/3proxy/.proxyauth
auth strong
allow alice
proxy -n -p10000 -a

# HTTP-прокси #2 (:10100) — открытый
flush
auth none
proxy -n -p10100 -a
```

### 🛡️ ACL — ограничение по IP

Можно разрешить подключение только с определённых адресов или подсетей (применяется глобально ко всем портам):

```
  ►  IP / подсеть (Enter — завершить): 1.2.3.4
  [ OK ]  Добавлен: 1.2.3.4
  ►  IP / подсеть (Enter — завершить): 10.0.0.0/8
  [ OK ]  Добавлен: 10.0.0.0/8
  ►  IP / подсеть (Enter — завершить):   ← пустая строка завершает
```

При включённом ACL все остальные адреса блокируются правилом `deny *`.

---

### ⚙️ systemd-сервис

```
[Service]
Type=simple
User=proxy3
Group=proxy3
Restart=on-failure
RestartSec=5
```

- Работает от отдельного системного пользователя `proxy3` (без shell, без домашнего каталога)
- Поднимается автоматически при старте сервера
- Перезапускается при падении

## 🔧 Управление сервисом

```bash
# Статус
systemctl status 3proxy

# Рестарт
systemctl restart 3proxy

# Остановить
systemctl stop 3proxy

# Логи в реальном времени
journalctl -u 3proxy -f

# Конфигурация
nano /etc/3proxy/3proxy.cfg

# Файл паролей
nano /etc/3proxy/.proxyauth
```

## 🔬 Проверка после установки

```bash
# Проверить открытые порты
ss -tlpn | grep 3proxy

# HTTP-прокси без авторизации
curl --proxy http://95.140.159.147:10000 https://ifconfig.me

# HTTP-прокси с авторизацией
curl --proxy http://alice:password@95.140.159.147:10000 https://ifconfig.me

# SOCKS5 без авторизации
curl --socks5 95.140.159.147:20000 https://ifconfig.me

# SOCKS5 с авторизацией
curl --socks5 bob:password@95.140.159.147:20000 https://ifconfig.me
```

## 📁 Структура файлов

```
/usr/bin/3proxy                         # Бинарник 3proxy
/etc/3proxy/
├── 3proxy.cfg                          # Основной конфиг (права 444)
└── .proxyauth                          # Файл паролей (права 400, только если есть хоть 1 авторизация)

/etc/sysctl.d/99-3proxy.conf            # Параметры ядра
/etc/iptables/rules.v4                  # Сохранённые правила iptables
/usr/local/src/3proxy/                  # Исходники (после компиляции)
```

## 🔐 Безопасность

- **Отдельный пользователь** `proxy3` — системный, без shell и домашнего каталога
- **MD5-хэш паролей** — пароли не хранятся в открытом виде (`user:CR:hash`)
- **Изоляция по пользователю** — `allow username` в каждом блоке: даже если пароли совпадают, пользователь имеет доступ только к своему порту
- **Права файлов** — конфиг `444`, файл паролей `400`
- **ACL + deny \*** — при включённом ограничении по IP все остальные блокируются
- **Валидация IP и портов** — скрипт проверяет корректность всех введённых значений, дубли портов исключены
- **iptables очищены** — перед стартом сервиса снимаются все лишние правила

## 🗑️ Удаление

```bash
systemctl stop 3proxy
systemctl disable 3proxy
rm -f /etc/systemd/system/3proxy.service
rm -rf /etc/3proxy /var/log/3proxy
rm -f /usr/bin/3proxy
userdel proxy3 2>/dev/null || true
systemctl daemon-reload
```

## 📝 Журнал изменений

**v6.0 — текущая версия**
- ✅ Авторизация на каждый порт отдельно — свой логин/пароль для каждого прокси
- ✅ Динамическое количество HTTP-прокси — скрипт предлагает добавить ещё один после каждого
- ✅ SOCKS5 опционален — спрашивает нужен ли он вообще, и тоже поддерживает несколько портов
- ✅ Защита от дублирования портов между HTTP и SOCKS5
- ✅ Единый файл паролей `.proxyauth` без дублей пользователей
- ✅ `allow username` в каждом блоке — изоляция пользователей по портам

**v5.0**
- ✅ `Type=simple` + `User=/Group=` вместо `Type=forking` + `PIDFile`
- ✅ Пароли хранятся как MD5-хэш (`CR`) вместо открытого текста (`CL`)
- ✅ `detect_ip()` через `ip route get` вместо `hostname -I`
- ✅ Валидация IP и портов при вводе
- ✅ iptables сбрасываются до старта сервиса
